1System Overview

The Sponsor Dossier Agent is a fully automated AI-powered research and outreach management system built for NorthStar AI Labs. It discovers potential sponsors across five categories — cloud/AI credit providers, Atlanta corporates, national tech companies, in-kind partners, and grants — plus a dedicated Small AI Startups track, finds decision-maker contacts at each company, drafts personalized outreach, scores every lead, and gives the team a full web dashboard to manage the pipeline from discovery through to a confirmed sponsor.

It has grown from a single-script research tool into a small full-stack web application: a Flask API backend, a React dashboard, deterministic lead scoring, per-contact outreach management, and a full settings layer for tuning how the agent searches and scores.

Architecture at a glance

No database, no queue — by design
  • Backend: Python 3 + Flask (flask-cors), single background thread per agent run
  • Persistence: flat JSON files only — settings.json, output/dossier.json, output/search_cache.json, output/photo_cache.json, output/api_usage.json, output/eligibility_cache.json
  • Frontend: React + Vite (dev server on :5173), react-router-dom, Tailwind CSS, plain fetch() against the Flask API — no state library, just React state + 2-second polling during a run
  • Everything runs synchronously within one background thread per run — intentionally simple, no vector DB, no task queue, no real database

3Core Agent Pipeline

End-to-End Flow
  1. User clicks Run Agent (Full Run) or Quick Refresh from the Dashboard
  2. Flask spawns a background thread running run_agent()
  3. GPT-4o-mini reasons over the 5 hardcoded categories + any custom keywords, deciding what to search
  4. SerpAPI/Brave executes company/program discovery searches
  5. Y Combinator directory (+ Crunchbase/Wellfound/Selenium fallback) discovers Small AI Startups separately
  6. Senior contacts (2/company) and mandatory Atlanta contacts (2/company) are found via LinkedIn-scoped search
  7. Stale/former-employee contacts are filtered out via title-pattern checks and a lightweight verification search
  8. A default outreach message is generated per contact automatically (template, not AI)
  9. Lead scores (company-level + per-contact) are calculated deterministically
  10. Results are written to dossier.json, Excel, and PDF; the dashboard updates in real time via polling

Full Run vs. Quick Refresh

  • Full Run — ignores the cache entirely and researches everything from scratch across all categories, Atlanta contacts, and custom keywords.
  • Quick Refresh — skips any company already cached and fresh (default 30-day window, configurable in Settings). Still searches for genuinely new companies, and for existing companies fills in specific missing gaps rather than re-researching the whole company. Logs cache hits vs. misses in the progress feed.

Search budget controls

Both search engines have independent, Settings-configurable budgets: a per-run cap (default 200/hour for SerpAPI, 300/run for Brave) and, where relevant, a monthly usage counter. When a run approaches either cap, the agent pauses and enters a paused_awaiting_confirmation state, offering "Finalize Now" or "Continue Anyway." Profile photo fetching is treated as a required category — the budget prioritizes discovery → senior contacts → Atlanta contacts → verification → photos, but photos are only skipped if the hourly cap is genuinely exhausted mid-run, never de-prioritized by design.

4Contact Discovery & Accuracy

Senior contacts + mandatory Atlanta contacts

Every company gets two contact searches run in parallel: the original senior/decision-maker search (VP, Director, Head of, or Founder-level for small companies) and a separate, mandatory Atlanta-scoped search using broadened, less senior role keywords (Marketing Manager, Community Manager, Regional Sales, Partnerships Coordinator, Program Manager) since local offices are less likely to have VP-level staff. The Atlanta search retries up to 2 phrasings if the first attempt returns nothing, then logs clearly if no local contact could be found rather than failing silently.

Each contact carries a location field — null for standard senior-search contacts, or a tagged value like "Atlanta, GA" for location-matched ones. The Results page has a dedicated Filter by Location control, separate from Filter by Status.

Stale / former-employee filtering

Contact scoring specifically detects and excludes people who no longer work at the target company — especially founders who've moved to a new venture while their LinkedIn title still reads "Founder" of the original company. Detection covers:

  • Direct phrasing: "former," "ex-," "previously at {company}"
  • Date-range titles suggesting a past, not current, role
  • Founder/Co-Founder titles where the snippet references a different, current company
  • Weak/absent current-role signals default to exclusion, not inclusion — accuracy over quantity

A lightweight secondary verification search runs only on the final selected contacts, not every candidate, to stay within the search budget.

5Lead Scoring Model

A deterministic, rule-based scoring system (no ML/training) built entirely on data the agent already collects, implemented in tools/lead_scoring.py. There are two separate scores — a company-level score and a per-contact responsiveness score.

5.1 Company-Level Lead Score (0–100)

FactorMaxLogic
Program/Offer Size30Parses offer text for dollar amounts and scale language. $50k+ → 30; $10k–50k → 24; $1k–10k → 18; under $1k → ~14. "Unlimited/uncapped" → 27. "Credits" with no figure → ~20. Vague text → 9. No offer text at all → 15 (neutral default).
Program Recency/Verification25Verified active → 25. Confirmed stale/discontinued → ~4. Missing/never checked → ~11 (uncertain, not penalized as heavily as confirmed-stale).
Category Fit25Configurable per-category weight: Credit Providers 25, Cash & In-Kind 25, AI Startup Sponsors 18, Small AI Startups 18, Grants 10 — credits/cash close faster than grants' longer application processes.
Contact Coverage20Full 20 if ≥1 Atlanta-tagged contact AND ≥1 senior contact. 12 if only one type. 5 if contacts exist but neither type. 0 if none found.

5.2 Per-Contact Responsiveness Score (0–100)

FactorMaxLogic
Role Type50Optimized for response likelihood, not seniority. Operational/high-intent roles (Partnerships, Community Manager, DevRel, Accelerator Lead) → 50. Founders at small companies → 50. Founders at large companies → 20. VP/Director/C-level at large companies → 15 (lowest). Same seniority at small companies → 30. No title on file → 20.
Location Match30Full 30 if tagged Atlanta, GA; 0 otherwise.
Recency of Verification20Full 20 if passed stale-contact verification. 8 if not run or inconclusive.

5.3 Where scores show up

  • Dashboard — color-graded score badge on every sponsor card (≥70 40–69 <40) plus "Sort by Lead Score"
  • Company Detail Page — full factor breakdown; contacts sorted by responsiveness, top contact labeled "Suggested first contact"
  • Outreach Portal — sortable Responsiveness column
  • Excel Export — Lead Score column plus per-contact Responsiveness column

Both scores recompute automatically on every dossier save — pure computation, no new searches. A "Recalculate All Scores" button in Settings re-scores everything after category weights are edited.

6Dashboard, Company Pages & Outreach Portal

Dashboard (Results Grid)

Each sponsor card shows only: logo, company name, program name, a colored status pill, a one-line offer summary, the lead score badge, and a "View Details (X contacts)" button — deeper detail lives on the company detail page. The status summary stat cards double as clickable filters, alongside a separate Filter by Location control. Each category has a distinct accent color and card tint.

Company Detail Page /company/:companyId

Shows the full offer and pitch text, an editable status dropdown, the full lead-score breakdown, and every contact — each with responsiveness score, LinkedIn link, location tag, editable outreach message with "Regenerate with AI," "Copy to Clipboard," and individually editable status.

Outreach Portal

A flat, sortable table of every contact across all companies — name, title, company, location tag, status, responsiveness score — backed by the same message data as the company detail page.

Small AI Startups — separate tab & exports

Small/early-stage AI companies (primarily sourced from the YC directory) live in their own dashboard tab with a dedicated PDF and Excel sheet. The main exports exclude this category entirely — every company appears in exactly one export.

Settings panel

  • Saved keyword libraries — separate lists for company-focus and contact-title keywords
  • Contacts per company — default 2 senior + 2 Atlanta = 4 target contacts
  • Seniority rules by company size — editable target-title lists
  • Cache freshness — default 30 days
  • Search budget caps — separate per-run and monthly caps for SerpAPI and Brave
  • Lead scoring weights — editable per-category Category Fit weights, plus "Recalculate All Scores"

7Excel & PDF Exports

Excel export

Main export has separate sheets for Credit Providers, Cash & In-Kind, AI Startup Sponsors, and Grants; Small AI Startups gets its own sheet/file. Columns include company info, program/offer text, lead score, and per-contact rows.

PDF export (ReportLab)

Restructured page order: one company info page, followed by one page per contact (photo, name, title, LinkedIn, location tag, full outreach message). Repeats per company, grouped by category. The Small AI Startups PDF uses the same structure, generated separately.

  • Company logos — Google Favicon API → DuckDuckGo icons → Brandfetch → generated placeholder
  • Contact photos — fetched via image search, circle-cropped with Pillow, cached per contact, initials fallback
  • Footer — every page shows the PDF generation date
  • Resilience — each contact's page generation is wrapped individually so one bad record can't crash the export

8Known Issues & Limitations

Search budget pressure

SerpAPI's 200/hour cap is tight relative to a full run's real search needs on a completely fresh, uncached run. Quick Refresh and caching substantially reduce this on repeat runs, but a first-ever full run may still hit the pause-and-ask flow.

Crunchbase / Wellfound scraping is effectively dead

Both sources are blocked at the robots.txt / bot-detection layer before a request is attempted, so the Selenium fallback rarely fires. The Y Combinator directory is the real, working source for Small AI Startups.

Profile photo accuracy

Photos are matched by name + company + title search, not verified against identity — the first usable image may occasionally be the wrong person. Visual verification before external use is recommended.

Lead scoring is rule-based, not machine-learned

The current model is a transparent, hand-weighted formula — by design, since there isn't yet enough historical outcome data to train a reliable classifier. This is Phase 1 of a longer plan: once enough real outcomes accumulate through status tracking, a lightweight ML model could be trained and blended in as Phase 2.

9Roadmap

Near term

  • ML-based lead scoring (Phase 2) — train a real model on accumulated outcome data
  • Similar-company suggestions — once a sponsor confirms, surface similar likely converters
  • News/trigger alerts — flag new program announcements or relevant funding news

Medium term

  • Multi-touch outreach sequences — a short follow-up sequence instead of one message
  • Follow-up reminders — flag contacts stuck in "Contacted" past a threshold
  • Team assignment — assign sponsors/contacts to specific team members
  • Pipeline dashboard — a funnel view (found → contacted → responded → confirmed)

Longer term

  • Scheduled runs + digest email — auto-run Quick Refresh and email a summary
  • Slack notifications — post when a sponsor is confirmed
  • Multi-tenant / workspace support — reuse the engine for other ventures via a company-switcher
